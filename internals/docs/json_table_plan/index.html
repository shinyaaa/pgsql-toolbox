PostgreSQL JSON_TABLE PLAN の内部構造

PostgreSQL 20devel · d15a6bc2e16 · 2026-07-15 00:51

JSON_TABLE は JSON ドキュメントをリレーションに変換する SQL/JSON 標準の機能である。 その中でも PLAN 句は、NESTED COLUMNS で表現される親子・兄弟のパスを どのように結合して行を組み立てるかを制御する、内部構造がもっとも凝縮した部分にあたる。 本ドキュメントは PostgreSQL 20devel（shinyaaa/postgres サブモジュール）のソースコードを 主たる典拠として、PLAN 句の構文・意味論から、パーサが構築する実行木、そして実行器が行を 1 件ずつ生成するアルゴリズムまでを、図と実例で解き明かす。想定読者は PostgreSQL の内部実装に 関心のあるエンジニアで、SQL/JSON の基本的な構文は既知とする。

第1章 JSON_TABLE と行生成モデル JSON_TABLE が JSON を行に展開する仕組みと、NESTED COLUMNS が PLAN を必要とする理由を示す。 第2章 PLAN 句の構文と意味論 明示 PLAN と PLAN DEFAULT の書き方、INNER/OUTER/UNION/CROSS の意味と既定を整理する。 第3章 PLAN を表す内部データ構造 未変換の JsonTablePlanSpec と、実行木 JsonTablePathScan / JsonTableSiblingJoin を対比する。 第4章 パーサによる PLAN の変換 transformJsonTableColumns と NESTED 列の再帰変換、および PLAN の妥当性検査を追う。 第5章 実行器の初期化 JsonTablePlanState 木の構築と、列番号からプラン状態への対応表 colplanstates を解説する。 第6章 行生成アルゴリズム JsonTablePlanNextRow による親子結合・兄弟結合の再帰と、OUTER/INNER/UNION/CROSS の実装を追う。 第7章 観察と検証 デフォルトプランの等価性、UNION と CROSS の違い、逆整形とバージョン上の注意を実例で確認する。
