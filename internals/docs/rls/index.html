PostgreSQL Internals: Row Level Security

PostgreSQL 19beta1 · dbaa4dc · 2026-06-29 10:46

本ドキュメントは PostgreSQL の 行レベルセキュリティ（Row Level Security, RLS） の内部実装を、ソースコードを引用しながら解説する技術ノートである。 カタログ pg_policy からクエリ書き換え経路、相補的なロール属性 （BYPASSRLS、FORCE ROW LEVEL SECURITY）、そして row_security GUC と plan invalidation の連動まで、RLS を構成する全要素を 6 章で概観する。

対象読者は PostgreSQL のバックエンド実装に関心のあるエンジニア・運用者で、 SQL レベルでの RLS 利用経験を前提とする。本書中の関数名・構造体名には GitHub 上のソースへのリンクを付している（fork: shinyaaa/postgres）。 扱う対象バージョンはページ上部のバッジに示すとおり、PostgreSQL 19beta1 系列の開発スナップショットである。

第1章 行レベルセキュリティの概観 RLS の目的と GRANT / 列レベル権限との違い、ポリシーの最小構成を整理する。 第2章 pg_policy カタログとデータ構造 ポリシーがディスクと relcache 上でどのように表現されているかを示す。 第3章 CREATE/ALTER/DROP POLICY の実装 DDL コマンドの実行経路と、依存関係・キャッシュ無効化の仕掛けを追う。 第4章 クエリ書き換え時の RLS 適用 クエリ書き換え段階で RLS がどのように合成され、Query ノードに注入されるかを示す。 第5章 ポリシー結合の意味論とコマンド別の挙動 permissive / restrictive ポリシーの結合規則と、SQL コマンド種別ごとの適用を整理する。 第6章 バイパス・FORCE・キャッシュ無効化の運用面 BYPASSRLS、FORCE ROW LEVEL SECURITY、row_security GUC、leakproof の絡みを実装に即して説明する。
