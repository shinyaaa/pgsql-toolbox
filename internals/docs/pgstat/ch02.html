第2章　統計種別とカインド抽象

pgstat.c は多様な統計を同じ枠組みで扱うため、統計の「種別（kind）」という概念を中心に据える。 種別ごとの振る舞いの違いは、1つのメタデータ構造体に集約される。 この章では、種別を識別する PgStat_Kind と、種別ごとの性質を記述する PgStat_KindInfo を解説する。

2.1　PgStat_Kind と種別ID

各統計種別には 1〜32 の整数IDが割り当てられる。IDは pgstat_kind.h で定義される。PGSTAT_KIND_MIN が 1、PGSTAT_KIND_MAX が 32 で、 ビルトイン種別は 1〜13、24以降がカスタム種別用に予約されている。

pgstat_kind.h（抜粋） 
#define PGSTAT_KIND_DATABASE      1   /* database-wide statistics */
#define PGSTAT_KIND_RELATION      2   /* per-table statistics */
#define PGSTAT_KIND_FUNCTION      3   /* per-function statistics */
#define PGSTAT_KIND_REPLSLOT      4   /* per-slot statistics */
#define PGSTAT_KIND_SUBSCRIPTION  5   /* per-subscription statistics */
#define PGSTAT_KIND_BACKEND       6   /* per-backend statistics */
#define PGSTAT_KIND_ARCHIVER      7
#define PGSTAT_KIND_BGWRITER      8
#define PGSTAT_KIND_CHECKPOINTER  9
#define PGSTAT_KIND_IO           10
#define PGSTAT_KIND_LOCK         11
#define PGSTAT_KIND_SLRU         12
#define PGSTAT_KIND_WAL          13

種別ID 1〜6 は可変数統計、7〜13 は固定数統計である。 PGSTAT_KIND_BUILTIN_MIN/MAX がビルトインの範囲、 PGSTAT_KIND_CUSTOM_MIN（24）以降が外部モジュール用である。 種別IDは PgStat_HashKey の一部として、また配列の添字として随所で使われる。

2.2　PgStat_KindInfo による抽象化

種別ごとの性質は PgStat_KindInfo 構造体にまとめられる。 これは pgstat.c が個々の種別の詳細を知らずに処理を回すための抽象化の要である。 主なフィールドを図 2.1 に示す。

フラグ群が種別の基本性質を決める。fixed_amount は固定数か可変数かを表し、 accessed_across_databases は他データベースから参照可能か（スナップショットに含めるか）を、 write_to_file は統計ファイルへ書き出すかを示す。 各種コールバックは種別固有の処理（フラッシュ、スナップショット構築、シリアライズ等）を プラグイン的に差し込む口であり、pgstat.c はこれらを通じてのみ種別固有処理を呼び出す。 種別とメタデータの対応付けは pgstat_get_kind_info() が行う。

2.3　ビルトイン統計の一覧

ビルトイン種別のメタデータは、pgstat.c 内の配列 pgstat_kind_builtin_infos に指定イニシャライザで定義される。 たとえば database 種別は次のように記述される。

pgstat.c（抜粋） 
[PGSTAT_KIND_DATABASE] = {
    .name = "database",
    .fixed_amount = false,
    .write_to_file = true,
    .accessed_across_databases = true,
    .shared_size = sizeof(PgStatShared_Database),
    .shared_data_off = offsetof(PgStatShared_Database, stats),
    .shared_data_len = sizeof(((PgStatShared_Database *) 0)->stats),
    .pending_size = sizeof(PgStat_StatDBEntry),
    .flush_pending_cb = pgstat_database_flush_cb,
    .reset_timestamp_cb = pgstat_database_reset_timestamp_cb,
},

ビルトイン種別を整理すると次のとおりである。

種別ID区分対象
database1可変数データベースごとの統計
relation2可変数テーブル／インデックスごと
function3可変数関数ごと
replslot4可変数レプリケーションスロットごと
subscription5可変数サブスクリプションごと
backend6可変数バックエンドごと
archiver7固定数WAL アーカイバ
bgwriter8固定数バックグラウンドライタ
checkpointer9固定数チェックポインタ
io10固定数I/O 統計（種別ごと）
lock11固定数ロック関連統計
slru12固定数SLRU キャッシュ
wal13固定数WAL 統計

2.4　カスタム統計種別

外部モジュールも独自の統計種別を定義できる。ビルトインと同じ性質をすべて利用でき、 他の拡張と衝突しないよう一意のIDを割り当てる必要がある。IDの予約は PostgreSQL Wiki の CustomCumulativeStats ページで管理される。

カスタム種別は起動時に pgstat_register_kind() で登録され、メタデータは pgstat_kind_custom_infos 配列（インデックスは kind - PGSTAT_KIND_CUSTOM_MIN）に格納される。 pgstat_get_kind_info() はビルトインかカスタムかを判定し、対応する配列から PgStat_KindInfo を返す。これにより、pgstat.c の処理ループは ビルトインとカスタムを区別せず一様に扱える。

関連種別ごとの共有メモリ表現とハッシュ格納は第3章で扱う。
