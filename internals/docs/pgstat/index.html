pgstat.c の内部構造

PostgreSQL 19beta1 · 031904048aa · 2026-06-21 08:53

PostgreSQL の累積統計システム（cumulative statistics system）は、テーブルアクセス数・チェックポイント回数・I/O 量といった稼働統計を収集し、pg_stat_* ビュー群を通じて提供する仕組みである。その基盤となるのが src/backend/utils/activity/pgstat.c であり、統計の共有メモリ表現・バックエンドローカルなキャッシュ・フラッシュ・スナップショット・ディスク永続化を束ねている。本ドキュメントは PostgreSQL 19beta1 のソースコードを基に、pgstat.c が実装する統計基盤のデータ構造と処理フローを図と実例で解説する。対象読者は PostgreSQL の内部実装に踏み込みたい開発者・運用エンジニアである。

第1章 累積統計システムの全体像 累積統計システムの役割、固定数／可変数の区分、ファイル分割、全体フローを俯瞰する。 第2章 統計種別とカインド抽象 統計種別 PgStat_Kind と、種別ごとの振る舞いを抽象化する PgStat_KindInfo を解説する。 第3章 共有メモリ上のデータ構造 PgStat_ShmemControl・dshash・HashKey など共有メモリ上のデータ構造を解説する。 第4章 ローカル参照とペンディング更新 ローカルハッシュ・PgStat_EntryRef・refcount／generation・ペンディングリストを解説する。 第5章 統計更新のフラッシュ pgstat_report_stat() を中心としたフラッシュ処理と頻度制御を解説する。 第6章 読み取りとスナップショット stats_fetch_consistency とスナップショットによる統計読み取りを解説する。 第7章 永続化とライフサイクル 統計ファイルの形式、書き出し・読み込み、リセットとライフサイクルを解説する。
