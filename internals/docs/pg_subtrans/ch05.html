第5章　ライフサイクル管理

本章では pg_subtrans の一生 — 初期化、起動、ページ拡張、チェックポイント、切り詰め — を順に たどり、最後に XID ラップアラウンドの扱いを見る。これらはすべて subtrans.c に実装される。

5.1　初期化と起動（BootStrap / Startup）

BootStrapSUBTRANS() は initdb 時に一度だけ呼ばれ、最初のページ（ページ 0）をゼロ初期化 してディスクへ書き出す（SimpleLruZeroAndWritePage()）。ディレクトリ自体は initdb のシェル処理が 作成済みである。

StartupSUBTRANS(oldestActiveXID) はサーバ起動時（StartupXLOG が nextXid を初期化した後）に一度呼ばれる。2.3節で述べた非永続性のため、 現在アクティブになりうるページ範囲 — oldestActiveXID のページから nextXid のページまで — をすべてゼロで初期化する。ラップアラウンドを考慮してページ番号が最大値を超えたら 0 へ巻き戻し、バンクロックは ページが属するバンクが変わるたびに取り直す。

5.2　ページ拡張（Extend）

新しい XID が新しいページにまたがるとき、そのページを使う前にゼロ初期化しておく必要がある。これを担うのが ExtendSUBTRANS(newestXact) である。XID 採番の途中（XidGenLock 保持下）で呼ばれるため、 できるだけ高速であることが求められる。図 5.2 のように、ページ先頭の XID（エントリ番号が 0、またはラップ直後の FirstNormalTransactionId）でないときは即座に戻り、何の I/O も発生させない。

5.3　チェックポイントと切り詰め（Checkpoint / Truncate）

CheckPointSUBTRANS() はダーティページをすべて書き出す（SimpleLruWriteAll()）。 2.3節のとおり、これは正当性のためではなく、書き込みをバックエンドから チェックポインタへ移すための最適化である。

TruncateSUBTRANS(oldestXact) はチェックポイント時にだけ呼ばれ、もはや参照されない古いセグメントを 削除する。oldestXact は実行中トランザクションの最古の TransactionXmin である。これを 1 だけ戻してからそのページを切り詰めの基準（cutoffPage）として SimpleLruTruncate() に 渡す。1 を引くのは、oldestXact がページ先頭かつ次XIDと一致する稀なケースで、まだ作られていない ページを基準に渡して SLRU のラップアラウンド検知を誤作動させないためである。

5.4　XID ラップアラウンドの扱い

XID は 32 ビットで循環するため、ページ番号やセグメント番号も 0xFFFFFFFF / 2048 付近で巻き戻る。 切り詰めの際に「どちらのページが古いか」を正しく判定するのが SubTransPagePrecedes() で、 CLOGPagePrecedes() と同様の発想で、ページ先頭XIDに FirstNormalTransactionId + 1 の オフセットを足してから TransactionIdPrecedes() で循環順序を比較する。

メモラップアラウンドの境界条件は誤りやすいため、起動時の SUBTRANSShmemInit() から SlruPagePrecedesUnitTests() が呼ばれ、 SubTransPagePrecedes() の整合性を単体テストで検証している。
