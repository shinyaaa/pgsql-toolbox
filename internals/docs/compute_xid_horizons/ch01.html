第1章　XID ホライズンの背景と必要性

PostgreSQL の追記型ストレージでは、行を更新・削除しても古いバージョン（タプル）は即座に消えず、 「もう誰からも見えない」と確定したときに初めて VACUUM が回収する。 その「誰からも見えない」を判定する境界が XID ホライズン（地平線, horizon） であり、 本章ではなぜこの境界が必要で、ComputeXidHorizons() がそこで何を担うのかを概観する。

1.1　MVCC と不要タプル

多版型同時実行制御（Multiversion Concurrency Control, MVCC）では、各タプルに挿入トランザクションの xmin と削除トランザクションの xmax が刻まれる。あるタプルが不要（dead）になるのは、 それを削除したトランザクションがコミット済みで、かつ 実行中のどのスナップショットからも見えなくなった ときである。逆に言えば、いま動いているトランザクションのうち最も古いものが見ている地点より新しい変更は、 まだ消してはならない。

図 1.1 に示すように、v1 の除去可否は「v1 を見うる最古のトランザクション」が存在するかどうかに帰着する。 この最古の地点を XID で表したものが地平線である。

1.2　下界としての oldest xmin

各バックエンドは自分のスナップショットの最小 xmin を MyProc->xmin に公示する （生きたスナップショットを持たないときは 0）。システム全体の地平線は、原理的には全バックエンドの 有効な xmin の最小値である。access/transam/README はこの計算が満たすべき正確性を次のように述べる。

READMEより ComputeXidHorizons は ProcArrayLock を共有ロックで取りながら有効な xmin の MIN() を取る。 その際、まだ xmin を設定していない開始直後のトランザクションが後から自分より小さい xmin を 設定してしまわないよう、実行中のすべての XID も MIN() の対象に含める。これにより、 並行する GetSnapshotData が後で見る最小 xmin を、こちらが過大評価することはない。

実行中トランザクションが一つも無い場合は latestCompletedXid + 1 を下界として用いる。 これは「これ以降に ProcArray へ現れうる XID の下限」であり、XidGenLock による インターロック（新しい XID は ProcArray へ格納してから可視になる）によって保証される。

1.3　GetSnapshotData が正確値をやめた理由

PostgreSQL 14 より前は、スナップショット取得 GetSnapshotData() のたびに正確な oldest xmin を 計算していた。しかし xmin は xid よりはるかに頻繁に変化するため、全バックエンドの xmin を毎回読むと キャッシュラインの奪い合い（ping-pong）が激しく、性能が出ない。そこで現在の GetSnapshotData() は 正確な地平線を計算せず、近似的な閾値だけを更新する（第4章）。正確な値が要るときにだけ ComputeXidHorizons() を呼ぶ、という二段構えになっている。

図 1.2 のとおり、ホライズン計算は「速いが粗い経路」と「遅いが正確な経路」に分離されている。 ComputeXidHorizons() は後者の中心にあり、VACUUM や pg_subtrans 切り詰めなど、 正確さが要る場面から呼び出される。

1.4　ComputeXidHorizons の位置づけ

ComputeXidHorizons() は storage/ipc/procarray.c に置かれた静的関数で、 単一の構造体 ComputeXidHorizonsResult に複数の地平線をまとめて書き込む。直接の呼び出し口は次の三つの ラッパ関数（および近似境界更新用の内部関数）である。

  • GetOldestNonRemovableTransactionId()　— VACUUM が、どのリレーションの不要タプルを保持すべきかを決める。
  • GetOldestTransactionIdConsideredRunning()　— pg_subtrans をどこまで切り詰めてよいかを決める。
  • GetReplicationHorizons()　— hot_standby_feedback で上流へ送る xmin / catalog_xmin を決める。

次章では、これらが取り出す ComputeXidHorizonsResult の各フィールドが何を意味するのかを一つずつ見ていく。
