共有バッファ（shared_buffers）の内部構造

PostgreSQL 19beta1 · 031904048aa · 2026-06-21 11:36

shared_buffers は、PostgreSQL がディスクページをメモリ上にキャッシュするための 共有バッファプールの大きさを決める設定である。本ドキュメントは、この一見ただの数値が PostgreSQL 内部でどのようなデータ構造として実体化し、ページの探索・置換がどう行われるかを、 PostgreSQL 19beta1 のソースコードに即して解説する。GUC 変数 NBuffers から始め、 バッファプールを構成する配列群、バッファ記述子 BufferDesc、バッファタグとハッシュ表、 そしてクロックスイープによる置換アルゴリズムまでを順に積み上げる。 PostgreSQL の内部に興味のあるエンジニアや、性能チューニングの背景を理解したい運用者を対象とする。

第1章 shared_buffers とは GUC 変数 NBuffers としての定義、単位の換算、共有メモリ上の位置づけを確認する。 第2章 バッファプールのメモリ構造 記述子・ページ本体・条件変数・チェックポイント領域の4配列と、その確保・対応づけ。 第3章 バッファ記述子 BufferDesc 各バッファのメタデータ。64bit の state に詰め込まれた refcount・usagecount・フラグ。 第4章 バッファタグとバッファテーブル ページを特定する BufferTag と、タグからバッファを引くハッシュ表・パーティションロック。 第5章 ページ探索と置換アルゴリズム ReadBuffer → BufferAlloc の流れと、フリーリスト廃止後のクロックスイープによる犠牲選択。 第6章 観察と運用 pg_buffercache・pg_shmem_allocations での観察と、shared_buffers サイズ設定の指針。
