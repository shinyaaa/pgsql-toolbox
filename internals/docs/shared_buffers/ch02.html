第2章　バッファプールのメモリ構造

「共有バッファプール」と一口に言うが、その実体はページ本体を格納する1つの配列だけではない。 各ページの状態を管理する記述子配列、I/O 待ちのための条件変数配列、チェックポイント用の ソート領域など、複数の配列が NBuffers を要素数として並んでいる。本章では、これらの 配列がどのように確保され、互いにどう対応づけられるかを見る。

実装は主に buf_init.c にある。

2.1　バッファプールを構成する4つの配列

バッファマネージャは、共有メモリ上に図 2.1 の4つの配列を確保する。いずれも要素数は NBuffers で、同じ添字 i が同じ1つのバッファスロットに対応する。

  • BufferDescriptors： 各バッファの記述子（メタデータ）の配列。どのページを保持しているか（タグ）や、 ピン数・使用回数・各種フラグといった状態を持つ。型は BufferDescPadded。第3章で詳説する。
  • BufferBlocks： ページ本体を格納する領域。NBuffers × BLCKSZ バイトの連続領域で、 これが shared_buffers のバイト数の主たる中身である。
  • BufferIOCVArray： バッファごとのI/O 完了待ち条件変数（ConditionVariable）の配列。 あるバッファへページを読み込んでいる最中、他プロセスが完了を待つために使う。
  • CkptBufferIds： チェックポイント時に、書き出すバッファ ID を並べ替えるための作業領域。 チェックポイント中の動的メモリ確保を避けるため、あらかじめ共有メモリに置く。
メモ記述子（メタデータ）とページ本体を別配列に分けるのは、状態だけを操作する 処理（ピン取得や状態フラグの確認）が、巨大なページ本体をキャッシュに載せずに済むようにするためである。 記述子は小さくキャッシュラインに整列され、頻繁な状態更新を効率化する。

2.2　共有メモリの確保

これらの配列は、起動時に2段階で準備される。まず BufferManagerShmemRequest() が必要なバイト数を申告し、続いて BufferManagerShmemInit() が確保された領域に対して各記述子を初期化する。図 2.2 にその流れを示す。

各領域の確保には整列（alignment）の指定が伴う。記述子と条件変数は PG_CACHE_LINE_SIZE 境界に、ページ本体は PG_IO_ALIGN_SIZE（I/O ページサイズ）境界に揃えられる。 ソース上の申告は次のようになっている。

buf_init.c（抜粋） 
ShmemRequestStruct(.name = "Buffer Descriptors",
                   .size = NBuffers * sizeof(BufferDescPadded),
                   .alignment = PG_CACHE_LINE_SIZE,   /* キャッシュライン整列 */
                   .ptr = (void **) &BufferDescriptors);

ShmemRequestStruct(.name = "Buffer Blocks",
                   .size = NBuffers * (Size) BLCKSZ,
                   .alignment = PG_IO_ALIGN_SIZE,     /* I/Oページ整列 */
                   .ptr = (void **) &BufferBlocks);

初期化ループ（図 2.2 下部）では、すべての記述子についてタグをクリアし、状態 state を 0、 そして buf_id に添字 i を設定する。buf_id はこのとき決まり、以後変化しない。

2.3　Buffer 番号と buf_id の対応

バッファを指すのに、内部では2つの番号体系が使われる。混同しやすいので整理しておく。

  • buf_id：配列の添字。0 〜 NBuffers-1 の0始まり。記述子・ブロック・条件変数の各配列をこの値で引く。
  • Buffer：バッファマネージャの外部 API が用いる番号。共有バッファは 1 〜 NBuffers の1始まりで、0 は InvalidBuffer を表す。

両者の関係は Buffer = buf_id + 1 である。図 2.3 にこの対応と、各番号から本体・記述子へ たどる流れを示す。GetBufferDescriptor(buf_id) は BufferDescriptors[buf_id] を、 BufferGetBlock(Buffer) は BufferBlocks 上の対応ページを返す。

負の Buffer 番号はローカルバッファ（一時テーブル用の、プロセスローカルなバッファ）を 表すが、本ドキュメントでは共有バッファに焦点を当てる。次章では、配列の中でも中心的な BufferDescriptors の各要素、すなわちバッファ記述子の内部を見ていく。
