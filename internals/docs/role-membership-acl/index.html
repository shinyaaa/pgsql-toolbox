PostgreSQL Internals: ロールメンバーシップと ACL 権限判定

PostgreSQL 20devel · d4046a48053 · 2026-07-22 13:40

PostgreSQL の権限判定は、ロール間のメンバーシップグラフを展開する層と、オブジェクトごとの ACL 配列をビットマスクで走査する層の二段構えになっている。本書は src/backend/utils/adt/acl.c を中心に、メンバーシップの再帰展開・プロセスローカルキャッシュ・ACL ビットマスク判定・GRANT の付与者選択、そして行レベルセキュリティ（RLS）でのロール判定までを、ソースコードを追いながら解説する。対象は PostgreSQL 20devel（コミット d4046a48053）。C 言語のバックエンドコードを読む前提知識がある読者を想定する。

第1章 ロールとメンバーシップの基礎 ロール・メンバーシップ・権限の三層構造と、判定関数の全体像を示す。 第2章 メンバーシップ展開の中核 roles_is_member_of メンバーシップグラフを幅優先で展開する中心関数を読む。 第3章 メンバーシップキャッシュの仕組み 1 スロットのプロセスローカルキャッシュと、その無効化・高速化の実装を追う。 第4章 判定関数の意味論 has_privs_of_role・member_can_set_role・is_member_of_role・is_admin_of_role の違いを明確にする。 第5章 ACL のビットマスク判定 aclmask Acl / AclItem のレイアウトと、権限ビットを集約する aclmask の三段構成を解説する。 第6章 GRANT の付与者選択 select_best_grantor 権限の出所を一意に保つための付与者選択ロジックを読む。 第7章 行レベルセキュリティへの適用 RLS がポリシーを選別する際に、どのロール判定関数を使うかを追う。
