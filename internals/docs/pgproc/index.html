PGPROC の内部構造

PostgreSQL 19beta1 · 0131e8fc508 · 2026-06-17 06:41

本ドキュメントは、PostgreSQL がプロセスごとに共有メモリ上へ確保する PGPROC 構造体の内部構造を、ソースコードを根拠に解き明かす技術解説である。PGPROC は、各バックエンドや補助プロセスが「いま何のトランザクションを実行し、どのロックを待ち、どのスナップショット地平を持つか」を他プロセスへ広告するための中核データ構造であり、MVCC・ロックマネージャ・スナップショット取得・同期レプリケーションといった PostgreSQL の根幹機能が ここに集約される。対象バージョンは PostgreSQL 19beta1。PostgreSQL のアーキテクチャに一通り触れたことのある開発者・運用者を想定読者とする。

第1章 PGPROCとは プロセスごとに共有メモリ上に確保される PGPROC 構造体の役割と、バックエンドとの対応関係を概観する。 第2章 PGPROC構造体のレイアウト PGPROC のフィールドを識別情報・トランザクション・シグナリング・ロックの観点で分類して読み解く。 第3章 ProcGlobal と共有メモリ配置 全 PGPROC を束ねる PROC_HDR、フリーリスト、そして密にパックされたミラー配列の設計を解説する。 第4章 PGPROCのライフサイクル 共有メモリ確保から割り当て・ProcArray 登録・解放までの一連の流れを追う。 第5章 PGPROCを介した協調動作 待機キュー・グループ XID クリア・ロックグループ・同期レプリケーションなど協調動作の実装を見る。 第6章 PGPROCを観察する pg_stat_activity や pg_locks など、PGPROC の状態を外から観察する方法を示す。
